🔧 COMO O CSS É CARREGADO
═══════════════════════════════════════════════════════════════════════════════

1. Site Estático (Porta 3000)
   └─ CSS servido normalmente da pasta raiz
   └─ Arquivos em: /css/
   └─ Bootstrap, jQuery, Custom CSS

2. Página de Login (via Proxy)
   └─ Next.js renderiza com CSS embutido
   └─ Tailwind CSS injetado
   └─ login.css importado localmente
   └─ Estilos inline para garantir aplicação

3. Proxy (Servidor Estático)
   └─ Não interfere com CSS
   └─ Headers corretos configurados
   └─ Cache desabilitado
   └─ Content-Type: text/html

4. Layout Root
   └─ Normalize CSS aplicado
   └─ Fonts importadas (Cairo, Montserrat)
   └─ Reset de margens/padding
   └─ Cores de background definidas


📄 VERIFICAÇÃO DO CSS
═══════════════════════════════════════════════════════════════════════════════

Para verificar se o CSS está carregando:

1. Abra o navegador:
   http://localhost:3000/admin/login

2. Abra DevTools (F12)

3. Verifique:
   ✓ Na aba Elements, veja as classes CSS aplicadas
   ✓ Na aba Network, verifique se login.css está sendo carregado
   ✓ Na aba Console, não deve haver erros 404
   ✓ Inspecione elementos com Ctrl+Shift+C

4. Elementos que devem estar estilizados:
   ✓ Fundo preto (#000000)
   ✓ Formulário com borda amarela
   ✓ Campos de input com estilos
   ✓ Botão com gradiente
   ✓ Texto em branco
