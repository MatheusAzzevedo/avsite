🎨 EXPLICAÇÕES DETALHADAS
═══════════════════════════════════════════════════════════════════════════════

Explicação da função [globals.css]:
   CSS global base que é carregado em TODAS as páginas do Next.js.
   Define reset CSS, importa fontes Cairo e Montserrat do Google Fonts,
   aplica Tailwind directives, define cores base (fundo preto, texto branco)
   e cria classes globais .admin-page e .login-page com !important para
   garantir fundo preto mesmo com estilos conflitantes.

Explicação da função [admin.css]:
   CSS específico para páginas administrativas (/admin/*).
   Define estilos compartilhados como .admin-header (cabeçalho com gradiente),
   .admin-btn-* (botões primary/danger/secondary), .admin-card (cards com
   bordas amarelas), .admin-table (tabelas com hover states), .admin-form-*
   (campos de formulário), e .admin-badge-* (badges de status).

Explicação da função [login.css]:
   CSS específico para a página de login (/admin/login).
   Define .login-page (container fullscreen com fundo preto), .login-form
   (formulário com gradiente e borda amarela), .login-field (grupo de label
   + input), .login-button (botão amarelo com hover animado), .login-error
   (mensagem de erro vermelha), e .login-decorative (círculos animados).

Explicação do Arquivo [layout.tsx]:
   Layout raiz do Next.js que envolve todas as páginas.
   Carrega o globals.css automaticamente e define a estrutura HTML base
   (<html>, <body>). Foi simplificado para remover estilos inline que
   conflitavam com o CSS global, garantindo que o fundo preto seja aplicado.

Explicação do Arquivo [admin/layout.tsx]:
   Layout específico para páginas administrativas.
   Carrega o admin.css e envolve o conteúdo com <div className="admin-page">
   para aplicar estilos compartilhados em todas as páginas /admin/*.

Explicação do Arquivo [admin/login/page.tsx]:
   Componente React da página de login.
   Renderiza o formulário de autenticação com campos de email e senha,
   botão de submit, mensagens de erro, e círculos decorativos animados.
   Foi limpo para remover estilos inline redundantes, permitindo que o
   CSS seja gerenciado apenas por arquivos CSS dedicados.

Explicação da Função [servidor-estatico.js]:
   Servidor Node.js HTTP que serve arquivos estáticos HTML do site Avoar
   e faz proxy de requisições /admin/* e /api/* para o Next.js na porta 3001.
   Usa http-proxy para manter o contexto das requisições, garantindo que
   CSS, JS e imagens sejam servidos corretamente. Isso permite que o site
   estático e o sistema administrativo Next.js coexistam harmoniosamente.

Explicação da Função [http-proxy]:
   Biblioteca Node.js que encaminha requisições HTTP de um servidor para outro
   mantendo headers, cookies e contexto. Usado no servidor-estatico.js para
   redirecionar transparentemente requisições /admin/* e /_next/* do localhost:3000
   para o Next.js em localhost:3001, permitindo que CSS e assets Next.js sejam
   carregados corretamente quando acessados via servidor estático.

Explicação da API [next.config.js rewrites]:
   Configuração do Next.js que mapeia rotas de entrada para rotas internas.
   As rewrites para /admin/:path* e /api/:path* garantem que quando o proxy
   encaminha requisições, o Next.js as processe corretamente, mantendo a
   estrutura de rotas e permitindo que assets como CSS sejam resolvidos
   nos caminhos corretos.
